# GOVERNANCE

Fluttrim 프로젝트의 의사결정 및 변경 승인 기준입니다.

## 유지보수 구조

- 프로젝트 오너/메인테이너가 최종 의사결정을 담당합니다.
- 외부 기여는 PR 기반으로 검토 후 병합합니다.

## 의사결정 원칙

- 안전성 우선: allowlist/containment 규칙을 훼손하는 변경은 반려
- 범위 준수: Flutter 관련 캐시/아티팩트 범위를 벗어나는 기능은 보류
- 재현 가능성: 동작 변경은 테스트와 문서 업데이트를 동반

## 변경 승인 기준

- 코드 변경: 최소 1회 리뷰(메인테이너) + CI 통과
- 정책/문서 변경: 운영 영향도 확인 후 병합
- 파괴적 변경: 릴리즈 노트/마이그레이션 안내 필수

## 릴리즈 원칙

- `packages/core`는 semver를 따릅니다.
- 공개 배포 전 `dart pub publish --dry-run` 검증을 필수로 수행합니다.
